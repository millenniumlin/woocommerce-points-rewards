# 會員等級管理升降級邏輯說明

## 概述

WooCommerce Points & Rewards 會員系統採用基於年度消費金額的等級制度，會員可以根據其年度累積消費自動升級或降級。

## 流程圖

```
用戶完成訂單付款
        ↓
更新年度消費統計
(update_user_yearly_stats)
        ↓
觸發等級升級檢查
(check_tier_upgrade)
        ↓
獲取用戶年度消費總額
        ↓
查找符合條件的最高等級
        ↓
    [有更高等級?]
         ↓      ↓
       是       否
         ↓      ↓
     執行升級   維持原等級
         ↓
   發送升級通知
         ↓
   設定過期時間(+1年)


每日執行等級過期檢查
        ↓
查找已過期的會員等級
        ↓
獲取當前年度消費總額
        ↓
重新計算符合的等級
        ↓
    [等級有變化?]
         ↓      ↓
       是       否
         ↓      ↓
   更新等級資訊  不做處理
         ↓
   觸發變更通知
         ↓
   重設過期時間(+1年)
```

## 快速參考

### 關鍵時機點

| 事件 | 觸發條件 | 執行動作 | 相關方法 |
|------|---------|---------|---------|
| 等級升級 | 訂單完成付款 | 檢查年度消費是否達到更高等級 | `update_user_yearly_stats()` → `check_tier_upgrade()` |
| 等級降級 | 每日檢查 + 等級過期 | 重新計算當前年度消費符合的等級 | `check_tier_expiry()` |
| 發送升級通知 | 等級升級完成 | 自動發送郵件通知用戶 | `send_tier_upgrade_notification()` |
| 等級變更通知 | 等級過期重新計算 | 觸發自定義通知處理 | `wc_points_rewards_tier_changed` Hook |

### 升級 vs 降級差異

| 比較項目 | 升級 | 降級 |
|---------|------|------|
| 觸發時機 | 訂單完成時即時檢查 | 每日檢查等級過期 |
| 執行頻率 | 每次消費後 | 每日一次 |
| 判斷依據 | 當年累積消費 | 當年累積消費 |
| 通知方式 | 自動郵件 | Hook 觸發 |
| 有效期設定 | +1年 | +1年 |

### 資料庫表結構

### 會員等級表 (`wc_points_rewards_tiers`)
- `id`: 等級唯一識別碼
- `name`: 等級名稱（如：微光會員、曙光會員、熾光會員）
- `min_amount`: 達到此等級所需的最低年度消費金額
- `bonus_percentage`: 該等級額外點數回饋百分比
- `tier_order`: 等級排序

### 用戶統計表 (`wc_points_rewards_user_stats`)
- `user_id`: 用戶ID
- `year`: 統計年份
- `total_spent`: 該年度累積消費總額
- `current_tier_id`: 當前等級ID
- `tier_start_date`: 等級開始時間
- `tier_expiry_date`: 等級過期時間（開始時間+1年）

## 升級邏輯

### 觸發時機
會員等級升級在以下情況觸發：
1. 用戶完成訂單付款時
2. 年度消費統計更新時（`update_user_yearly_stats` 方法）

### 升級流程
1. **消費記錄更新**
   ```php
   // 當用戶完成訂單時，更新年度消費統計
   $this->update_user_yearly_stats($user_id, $order_amount);
   ```

2. **等級檢查**
   ```php
   // 自動觸發等級升級檢查
   $this->check_tier_upgrade($user_id, $year);
   ```

3. **升級條件判斷**
   - 系統查詢用戶當年累積消費總額
   - 尋找符合條件的最高等級（`min_amount <= 用戶消費總額`）
   - 按 `min_amount` 降序排列，取第一個

4. **升級執行**
   - 使用資料庫鎖定機制防止競爭條件
   - 更新用戶統計表中的等級資訊
   - 設定等級開始時間為當前時間
   - 設定等級過期時間為開始時間+1年
   - 觸發 `wc_points_rewards_tier_upgraded` 動作

5. **通知發送**
   - 自動發送升級通知郵件給用戶
   - 郵件包含新等級名稱和額外回饋百分比

### 升級範例
假設會員等級設定如下：
- 微光會員：年消費 ≥ 5,000 元，額外回饋 10%
- 曙光會員：年消費 ≥ 10,000 元，額外回饋 20%
- 熾光會員：年消費 ≥ 20,000 元，額外回饋 30%

用戶消費軌跡：
1. 用戶年初消費 3,000 元 → 維持一般會員
2. 用戶累積消費達 6,000 元 → 自動升級為微光會員
3. 用戶累積消費達 12,000 元 → 自動升級為曙光會員
4. 用戶累積消費達 25,000 元 → 自動升級為熾光會員

## 降級邏輯

### 觸發時機
會員等級降級在以下情況觸發：
1. 每日自動檢查（`wc_points_rewards_daily_cleanup` 鉤子）
2. 等級過期時重新計算

### 降級流程
1. **過期檢查**
   ```php
   // 每日執行，檢查過期的會員等級
   $this->check_tier_expiry();
   ```

2. **過期等級識別**
   - 查詢 `tier_expiry_date <= NOW()` 的所有會員記錄
   - 確認等級確實已過期

3. **重新計算等級**
   - 獲取用戶當前年度的消費總額
   - 根據當前消費金額重新計算符合的等級
   - 如果當前消費不足以維持原等級，則降級

4. **降級執行**
   - 更新用戶的 `current_tier_id`
   - 重設 `tier_start_date` 為當前時間
   - 重設 `tier_expiry_date` 為新開始時間+1年
   - 觸發 `wc_points_rewards_tier_changed` 動作

### 降級範例
用戶狀況：
- 2023年累積消費 25,000 元，升級為熾光會員
- 熾光會員等級於 2024年過期
- 2024年累積消費僅 8,000 元

降級過程：
1. 系統檢測到該用戶的熾光會員等級已於 2024年過期
2. 重新計算：2024年消費 8,000 元，符合微光會員條件（≥5,000元）
3. 用戶被降級為微光會員
4. 新的微光會員等級有效期為 2024年開始+1年

## 特殊機制

### 1. 競爭條件防護
```php
// 使用資料庫鎖定防止同時處理
$lock_name = "wc_points_tier_upgrade_{$user_id}_{$year}";
$lock_acquired = $wpdb->get_var($wpdb->prepare("SELECT GET_LOCK(%s, 10)", $lock_name));
```

### 2. 預設等級處理
- 當用戶沒有等級或等級過期時，系統會指派 `tier_order = 1` 的預設等級
- 預設等級通常是最低等級，不需要最低消費要求

### 3. 年度重置機制
- 每年度消費統計獨立計算
- 跨年時，用戶需要重新累積消費來維持或升級等級
- 等級過期時間以等級獲得時間+1年計算，而非以年度為單位

## 程式碼位置

### 主要檔案
- `includes/class-database.php` - 升級邏輯實現
- `includes/class-member-tier.php` - 降級邏輯和等級管理
- `admin/views/tier-edit.php` - 等級管理介面
- `frontend/views/account-member-tier.php` - 前端等級顯示

### 關鍵方法
- `update_user_yearly_stats()` - 更新年度消費並觸發升級檢查
- `check_tier_upgrade()` - 執行等級升級邏輯
- `check_tier_expiry()` - 執行等級過期和降級檢查
- `get_user_current_tier()` - 獲取用戶當前有效等級

## 通知機制

### 升級通知
- 觸發時機：用戶升級時
- 通知方式：系統郵件
- 內容包含：新等級名稱、額外回饋百分比
- 可擴展：透過 `wc_points_rewards_tier_upgraded` 鉤子

### 降級通知
- 觸發時機：等級過期重新計算後
- 通知方式：透過 `wc_points_rewards_tier_changed` 鉤子
- 可自定義：開發者可監聽此鉤子實現自定義通知邏輯

## 管理員功能

### 等級設定
- 可在後台管理介面新增、編輯、刪除會員等級
- 設定最低消費金額和額外回饋百分比
- 查看各等級的會員數量和統計資訊

### 手動調整
- 管理員可以手動調整用戶的會員等級
- 可查看用戶的等級變更歷史
- 支援批量處理歷史訂單以重新計算等級

這個系統確保了會員等級的公平性和自動化管理，同時提供了足夠的靈活性來滿足不同的業務需求。
