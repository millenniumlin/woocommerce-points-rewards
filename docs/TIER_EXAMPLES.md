# 會員等級升降級實例說明

## 實際案例演示

假設系統中設定了以下會員等級：

| 等級名稱 | 最低年消費 | 額外回饋 | 等級順序 |
|---------|-----------|---------|---------|
| 微光會員 | 5,000 元  | 10%     | 1       |
| 曙光會員 | 10,000 元 | 20%     | 2       |
| 熾光會員 | 20,000 元 | 30%     | 3       |

## 案例一：會員升級過程

### 用戶 A 的消費軌跡（2024年）

**1月份：**
- 訂單金額：3,000 元
- 累積消費：3,000 元
- 會員等級：一般會員（未達到任何等級門檻）
- 系統動作：記錄消費，檢查升級條件，無符合條件的等級

**3月份：**
- 訂單金額：2,500 元
- 累積消費：5,500 元
- 會員等級：**自動升級為微光會員**
- 系統動作：
  ```php
  // 觸發升級檢查
  update_user_yearly_stats(user_id: 123, amount: 2500, year: 2024)
  -> check_tier_upgrade(user_id: 123, year: 2024)
  -> 查詢符合條件的最高等級：微光會員（5,000 <= 5,500）
  -> 更新等級，設定過期時間為 2025年3月
  -> 發送升級通知郵件
  ```

**8月份：**
- 訂單金額：5,000 元
- 累積消費：10,500 元
- 會員等級：**自動升級為曙光會員**
- 系統動作：檢查發現符合曙光會員條件，自動升級

**12月份：**
- 訂單金額：12,000 元
- 累積消費：22,500 元
- 會員等級：**自動升級為熾光會員**
- 系統動作：檢查發現符合熾光會員條件，自動升級至最高等級

## 案例二：會員降級過程

### 用戶 B 的等級變化

**2023年狀況：**
- 年度消費：25,000 元
- 達成等級：熾光會員
- 等級有效期：2024年12月31日

**2024年狀況：**
- 年度消費：8,000 元（消費減少）
- 等級過期：2024年12月31日系統檢查

**2025年1月1日系統檢查：**
```php
// 每日過期檢查執行
check_tier_expiry()
-> 發現用戶B的熾光會員已過期
-> 獲取2024年消費：8,000元
-> 重新計算符合等級：微光會員（5,000 <= 8,000 < 10,000）
-> 更新為微光會員，重設有效期至2026年1月1日
-> 觸發等級變更通知
```

## 案例三：跨年度等級變化

### 用戶 C 的年度轉換

**2023年底：**
- 年度消費：18,000 元
- 會員等級：曙光會員
- 等級過期：2024年某月

**2024年初：**
- 年度消費重新開始計算：0 元
- 會員等級：仍為曙光會員（未過期）

**2024年等級過期時：**
- 當前年度消費：6,000 元
- 重新計算：符合微光會員條件
- 等級變化：曙光會員 → 微光會員（降級）

## 系統特性說明

### 1. 即時升級
- 消費達到門檻立即升級
- 不需要等待任何審核期間
- 自動發送升級通知

### 2. 延遲降級
- 只有在等級過期時才會降級
- 給予會員一年的維持期間
- 基於當前年度消費重新評估

### 3. 防重複處理
- 使用資料庫鎖定機制
- 防止同時處理造成的錯誤
- 確保資料一致性

### 4. 靈活的通知機制
- 升級時自動發送郵件
- 支援自定義通知處理
- 透過 WordPress Hook 擴展

這套系統確保了會員等級的公平性，同時鼓勵持續消費以維持等級優勢。
